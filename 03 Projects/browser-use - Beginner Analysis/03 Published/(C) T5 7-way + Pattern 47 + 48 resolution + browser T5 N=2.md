## Part 3 — Pattern #47 Vision-Based Browser Automation RESOLUTION

### Prior history

- **v24 Skyvern registration (N=1):** "Vision + LLM replaces DOM/XPath"
- **v29 crawl4ai refinement:** narrowed to "vision-as-primary architectural alternative"

### v41 browser-use evidence

**Source-level verification** from `browser_use/dom/` + `browser_use/agent/service.py`:

1. `dom/service.py` = 1,081 LOC, 45.1KB — major DOM subsystem
2. `SerializedDOMState + ClickableElementDetector + EnhancedDOMTreeNode` — rich DOM representation
3. Agent assigns **numeric indices to clickable elements** (DOM-based referencing: `browser-use click 5`)
4. `use_vision: bool | Literal['auto'] = True` — vision enabled but **screenshot tool excluded when `use_vision != 'auto'`** (opt-out, not integral)
5. `highlight_elements=True` Browser param — **DOM drives visual markup in screenshots** (DOM is primary signal; vision annotates DOM-identified elements)

**Architectural conclusion:** DOM tree + AX-tree + element-indexing is PRIMARY; vision screenshot AUGMENTS DOM for disambiguation (visually-similar elements / canvas-rendered text / image-based content).
