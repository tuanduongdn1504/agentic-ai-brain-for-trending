The transferable wisdom — failures first-class

why-we-killed-ci-sweeper

/loop 5m on red main → ~8M tokens in 48h, 11 PRs, one broke prod config. Root causes: no L1 phase · verifier in the same session · no branch allowlist · no daily budget. The 4-point cascade is the pre-flight checklist for any auto-fix loop.

Verifier parity

A verifier APPROVEd a dependency patch after cached npm test; CI’s npm ci failed on a peer conflict. Lesson: the verifier must run the exact same install path as CI.

Multi-loop collision

Two loops fixed the same PR five minutes apart — ~400k tokens vs a normal ~80k. Fix: branch locks + an acting_on collision check across state files.

L1→L2 graduation bar

Two weeks at L1 · <20% noise · verifier proven manually first · denylist documented · loop-audit ≥58. “Report-only is not wasted time — it calibrates triage.”

9 failure modes (infinite fix loop, state rot, verifier theater, token burn, parallel collision…) + 10 anti-patterns (same agent verifies own work · no attempt cap · L3 before L1 · no kill switch · auto-merge without allowlist · no run log…).
