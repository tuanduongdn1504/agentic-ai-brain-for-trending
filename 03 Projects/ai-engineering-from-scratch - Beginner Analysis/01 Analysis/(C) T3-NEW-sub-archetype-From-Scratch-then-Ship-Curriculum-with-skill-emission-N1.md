## Claim

Register a NEW **T3 Education** sub-archetype, **PROVISIONAL N=1**:

> **"From-Scratch-then-Ship Curriculum with agentskills.io Artifact Emission"** — a curriculum whose pedagogical *output unit is a deployable agent-artifact*. The lesson spine is engineered to terminate in a "Ship It" step that emits a reusable prompt / agentskills.io-format SKILL.md / agent loop / MCP server, and the repo itself ships operational agentskills.io skills + an installer + scaffolded artifact-output directories.

## Why a NEW sub-archetype (typological distinction vs the 3 prior T3 Education subjects)

T3 Education has 3 prior corpus anchors. v113 is structurally distinct from all three:

|---|---|---|---|

The load-bearing distinction is the **"Ship It" spine**. Per the README's 6-step lesson pattern (Motto → Problem → Concept → Build It → Use It → **Ship It**) and AGENTS.md (*"Each lesson ships a reusable artifact you can plug into your daily workflow"*), the curriculum's deliverable is not knowledge-in-your-head (v74/v77) nor an importable library (v111) — it is a **portfolio of deployable agent-artifacts** in the same format the rest of the corpus's agentskills.io chain uses (v76 → v93 → v98 → v99 → v100).

## Structural criteria (sub-archetype definition — 5 properties)

1. **From-scratch-before-framework pedagogy** — Build It (raw math, no framework) precedes Use It (production library). [README + AGENTS.md "the spine".]
2. **Artifact-emitting "Ship It" terminal step** — every lesson's deliverable is a reusable prompt / skill / agent / MCP server.
3. **Ships operational agentskills.io skills in-repo** — `.claude/skills/find-your-level/SKILL.md` + `.claude/skills/check-understanding/SKILL.md`, real YAML frontmatter (`name`/`version`/`description`+trigger-phrases/`tags`).
4. **Artifact installer + scaffold** — `scripts/install_skills.py`; `outputs/{skills,agents,mcp-servers,prompts}/` directories + `index.json` catalog.
5. **AGENTS.md operating manual for AI agents** — the repo is authored to be consumed by AI agents as well as humans (Library-vocab #12 LLM-routing artifact).

All 5 PASS at source (verified against repo tree + README + AGENTS.md + SKILL.md).

## HONEST CAVEAT (load-bearing — do not bury)

The sub-archetype is registered on the **pedagogical-structure** distinction, **NOT** on a claim of 473 shipped artifacts:

- `outputs/skills/` contains only **`.gitkeep`** — it is an empty scaffold. The 473 "Ship It" artifacts are **student-generated by design** (the curriculum produces them *in the learner's hands*), not committed to the repo.
- Only **2** agentskills.io skills are actually realized in-repo, and both are **meta-skills about the curriculum** (placement quiz + per-phase quiz), not domain artifacts from the lessons.

So the structural claim is precise: *a curriculum architected to emit deployable agent-artifacts, with a working 2-skill proof-of-concept + installer + scaffold* — not *a 473-artifact library*. Pattern #82 quantitative-marketing ("473 artifacts you authored") is exactly that: a marketing frame for student output, pressure-tested and labeled here.
