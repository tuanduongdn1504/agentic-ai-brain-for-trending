## Sub-archetype definition (provisional)

A **T1 Assistant Skill bundle** structured as N distinct skills (typically **8-15**), each addressing a specific design/aesthetic-preference or workflow-sub-mode, unified by:
1. A **single brand identity** (in v81: the "taste-skill" brand + custom `tasteskill.dev` domain).
2. A **setting-based spectrum-control mechanism** (dimensional sliders / tiers / qualifiers tuning each skill within shared dimensions).
3. **Packaged for portable multi-harness installation** via `npx skills add` or equivalent skill-CLI install vectors.
4. **Sub-variant taxonomy decomposable along orthogonal axes** (in v81: 3 axes × 4 skills/axis = 4 aesthetic + 4 workflow + 4 platform-extension).

## Distinction from v75 impeccable "Single-Skill Multi-Target Distribution"

Both v75 + v81 fall under T1 Assistant Skill / design-language tier. The distinction is at **layer of multiplicity**:

| Layer | v75 impeccable | v81 taste-skill |
|---|---|---|
| **Skill content** | 1 canonical skill | 12 distinct skills |
| **Harness distribution** | 14 harness directory replicas | ~4 harness ecosystem coverage |
| **Multiplicity location** | Distribution layer (harness × 14) | Content layer (skills × 12) |
| **Sub-variant axis** | Harness-target (provider/IDE) | Aesthetic + workflow + platform-extension |
| **User install decision** | One skill, one install | 12 separate install decisions per `--skill` flag |

**Conclusion**: **Architectural inversion at layer-of-multiplicity dimension**. v75 = multi-target distribution; v81 = multi-skill content. Same tier, structurally orthogonal sub-archetypes.
