# (C) Cluster: User-facing — what easy-vibe teaches + how learners use it

## In one sentence

easy-vibe is a CC BY-NC-SA 4.0 vibe-coding curriculum from **Datawhalechina** (Chinese AI/ML open-source education community) + Tsinghua University contributors — 4 progressive stages teaching beginners to build AI-enabled apps through dialogue with AI coding tools (Claude Code + Cursor + Trae + VS Code AI + Dify), with 120+ interactive topics + 13 languages + AI-Agent-optimized navigation via `llms.txt`.

## The methodology: "vibe coding"

> "Jump right in and vibe together — if you can talk, you can build apps."

The course inverts traditional learning order:

| Traditional coding | Vibe coding (easy-vibe) |
|---------------------|-------------------------|
| Learn syntax + theory → build later | Build first → understand mechanisms after |
| Deep technical onboarding before producing | Conversational specification → AI generates |
| Single-tool mastery | Multi-tool dialogue fluency |
| Code-as-product | Conversation-as-product |

"In the AI era, programming starts by describing what you want."

This is **CORPUS-FIRST explicit PRO-VIBE positioning** — prior corpus subjects treated vibe coding as background context or critiqued it. easy-vibe names it as the affirmative methodology and builds a 120+ topic curriculum around it.

## 4-stage curriculum

| Stage | Audience | Outcome |
|-------|----------|---------|
| **Stage 0 ("Kindergarten")** | Absolute beginners | First conversational interactions with AI coding tools |
| **Stage 1 - Beginner Entry** | PMs / Founders / Aspiring builders | Idea validation + prototyping + AI capability integration |
| **Stage 2 - Full-Stack Development** | Junior developers | Frontend design + backend APIs + databases + deployment + payment systems |
| **Stage 3 - Advanced Development** | Senior developers | Claude Code workflows + MCP + Skills + agent teams + cross-platform delivery |
