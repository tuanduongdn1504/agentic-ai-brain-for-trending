## 4. getdesign.md commercial model — Pattern #50 candidate

### Commercial model

- GitHub repo (awesome-design-md) = MIT OSS (free content)
- **getdesign.md** = commercial companion website
  - Preview catalog + search
  - Sponsorship slots ("logos on that platform")
  - Featured listings
  - Possibly premium content

### Distinct from Pattern #31 open-core

| Pattern | Structure | Example |
|---------|-----------|---------|
| **#31 Open-Core** | OSS product + proprietary tier on SAME product | fish-speech commercial license, Skyvern Cloud anti-bot |
| **#50 Commercial-Funnel Companion** | OSS content + SEPARATE commercial platform | awesome-design-md GitHub + getdesign.md sponsorship |

### Pattern #50 registration

> **Pattern #50 Commercial-Funnel Companion Platform (CANDIDATE N=1):** Content aggregators or OSS projects use commercial companion platform (separate website with sponsorship / paid features / premium listings) as monetization channel. Distinct from Pattern #31 open-core (proprietary tier on same product). Commercial platform typically hosts preview / catalog / showcase + monetized slots. Cross-drives attention: OSS virality → platform traffic → commercial revenue.

**Evidence:** VoltAgent's getdesign.md.

**Validation path:** Monitor awesome-list-genre projects + content aggregators for similar commercial-funnel companion structures.
