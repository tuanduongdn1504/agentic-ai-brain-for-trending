What it is

A research team, expressed as skills

AI Berkshire systematizes four value-investing masters into an installable Claude Code / Codex skill collection — 18 skills, 8 Python tools, and a documented three-layer architecture.

Its thesis is a jab at vanilla prompting: ask an LLM “should I buy this?” and you get a both-sides hedge that ends in “do your own research.” AI Berkshire forces the opposite — a Pass / Gray / Fail verdict with concrete price ranges and tiered recommendations. The engine that produces it is worth studying whether or not you ever buy a share.

The transferable part on-goal

Six agent-engineering patterns worth stealing

The reason this belongs in your wiki: it’s a concrete, readable blueprint for the multi-agent workflows you’re already piloting. The finance domain is incidental.

1 · Persona-differentiated parallel orchestration
Four general-purpose subagents launched in one message, each a lens, reporting to a synthesizing lead. Same input, different framing. Your multi-agent-orchestration pilot thread, made concrete.
2 · Deterministic-tool-offload
LLMs can’t be trusted with a P/E to one decimal — so arithmetic is handed to decimal.Decimal Python as a mandatory gate. The single most portable idea here: make the model call code for what it guesses wrong.
3 · A verification / exit gate before “publish”
report_audit.py samples 15% of a report’s numbers, re-verifies each at 1% tolerance, and returns PASS/FAIL with a CI exit code. Your own verify-rule, productized.
4 · Cross-harness distribution from one source
One canonical skills/*.md → generated Codex skills + prompts via a sync script with a --check drift mode. Never hand-edit generated files.
5 · Forced-verdict discipline
The output shape (scorecards, decision memos, price tiers) is built into the skill so the model can’t hedge. The “mirror test”: can’t state it in 5 sentences → don’t buy.
6 · Structured anti-bias mechanisms
A/B/C information-richness rating · Munger inversion · quick-kill vetoes · banned phrases (“I think”, “obviously”) · honest “I don’t know” over speculation.
The knowledge

Kelly, moats, and the mirror test

The durable frameworks (the stock picks are time-bound; these aren’t). The Kelly formula is the spine — the math of survival, not maximum growth:

Kelly sizing — overbetting kills you even when every bet has positive expected value.

The mirror test

Complete each in one clear sentence or don’t buy: (1) I understand the cash machine · (2) the moat is ___, widening/stable · (3) management is trustworthy because ___ · (4) price = ___% of fair value · (5) if I’m wrong, downside is controllable because ___.

The moat test: “If I gave a competitor $10B, could they replicate this in five years?” Yes → the moat is too narrow. The five buckets: brand/pricing-power · switching costs · network effects · scale/cost · tech/patents. Screen by exclusion, not by ranking — kill the definitely-not-first-tier, deep-dive the rest.

Honest caveats

Unverified

The +69.29% / +66.38% / ¥1.46M returns have no trade logs — only brokerage screenshots. The repo’s own memory file admits the README’s sample outputs are illustrative/fictional. Treat performance as unproven; this is not investment advice (the repo says so — DYOR).

“Four masters arguing” is framing — it’s four parallel persona-instructions synthesized by a fifth, not a true debate. Li Lu’s long-run alpha is overstated in retail lore (the disclosed Munger investment ≈ 10–11% CAGR ≈ the S&P; BYD was the real home-run). Footgun: the README suggests claude --dangerously-skip-permissions — don’t; approve tool calls.
