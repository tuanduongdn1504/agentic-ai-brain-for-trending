## 3. Surprises + corrections

### Surprise 1: 2-day-old project with corpus-2nd-highest velocity

Initial assumption: programming language launches are usually slow (compiler infrastructure + tooling + ecosystem takes months). Zero v68 disrupts this — 91 commits + 3 releases + 2,100 stars in 2 days = SUBSTANTIAL-from-day-one launch.

CHANGELOG v0.1.0 entry confirms: *"Initial public release of Zero as the programming language for agents. Includes the native compiler, examples, documentation site, and validation fixtures."* The v0.1.0 release wasn't an "MVP teaser" — it shipped with native compiler + docs + examples + validation.

**Lesson captured:** Corporate-strategic-lab launches (Vercel Labs scale) can ship substantial-from-day-one + drive viral velocity. Distinct from typical-MVP-launch patterns. Add to v69 Pattern #52 sustained-velocity-test as TEST CASE for "corporate-substantial-launch sustains velocity?" question.

### Surprise 2: SKILL.md is intentionally thin (corpus-first mechanism)

Initial expectation: SKILL.md would be the full skill content. Zero v68's SKILL.md (51 lines) explicitly says: *"This file is a discovery stub. Do not treat it as the full Zero workflow."* The full content is served by the compiler binary at runtime.

CHANGELOG v0.1.1 confirms this design choice: *"Keeps the installable Zero skill as a thin bootstrap so external skill managers discover one Zero skill while the compiler serves the richer guidance for the installed version."*

**Lesson captured:** Skill-content-version-drift is a real problem in agentic ecosystems. Zero v68's Skill-As-Binary-Bootstrap is corpus-first solution. Mechanism worth observing for generalizability — add to OC observational track.

### Surprise 3: Rust-style borrow checker as language feature

Initial expectation: 2-day-old language won't have sophisticated semantic analysis. CHANGELOG v0.1.2 contradicts: *"Rebuilds borrow provenance tracking across references, fields, subpaths, assignments, control-flow joins, receiver side effects, generic methods, return summaries, and unreachable paths."*

**Lesson captured:** Zero is positioned as systems-language (memory safety via borrow checking), not scripting language. Corpus-first observation — no prior corpus subject documented borrow-checker semantics.

**Initial assumption:** Zero v68's CHANGELOG named contributors per release (e.g., `@ctate @badlogic @chenrui333 @heylakatos @onevcat` for v0.1.2) might be N=2 evidence for Pattern #82 Gamified-Curated Community Contribution.
