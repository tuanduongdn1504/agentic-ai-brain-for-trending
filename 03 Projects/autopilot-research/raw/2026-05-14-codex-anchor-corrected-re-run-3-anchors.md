## 5. Takeaways

Based on the sources, here are nine actionable rules and configurations for 
developers adopting agentic workflows with Codex and Claude Code:

1.  **Enable Autonomous "Goals":** To execute long-running tasks without manual 
intervention, manually enable the experimental feature by adding `features.goals
= true` to your `config.toml` file [1]. This allows the agent to run for hours 
or even days to solve complex problems through an integrated "Ralph loop" [2, 
3].
2.  **Establish a "Northstar" in Plan Mode:** Always use "Plan Mode" before 
starting a goal to define extremely specific, quantifiable completion criteria 
[4]. Chase AI emphasizes that without this specific end state, the AI’s 
autonomous output may be "mediocre" or "half-baked" [4, 5].
3.  **Adopt Tool Agnosticism:** Run the Claude Code terminal inside the Codex 
desktop app to use both tools in tandem rather than choosing one [6, 7]. This 
allows you to perform "adversarial reviews," where one model (like Claude) 
critiques the plan or code generated by the other to catch missed bugs [8, 9].
4.  **Lazy-Load MCP Tools:** In your configuration, set 
`tool_search_always_defer_mcp_tools = true` to enable lazy-loading for MCP 
servers [10]. Tù Bà Khuỳm notes this prevents wasting thousands of tokens by 
only loading tool definitions when they are actually needed [10].
5.  **Isolate Tasks with Sub-Agents:** Use the "Multi-agent" configuration to 
spawn independent sub-agents for specific roles like "QA Reviewer" or "Shell 
Builder" [11]. This keeps the main agent’s context "clean" and allows for 
parallel execution in separate sessions [12, 13].
6.  **Rewind to Fix Hallucinations:** Instead of arguing with an agent when it 
goes off-track, use the "Rewind" feature (double-tap ESC) to discard the failed 
turns and edit the previous prompt [14]. This restarts the session with 
corrected information and removes the "garbage" turns from the context [14].
7.  **Require Automated Verification Scripts:** Include a mandatory step in your
goal criteria for the agent to write and run automated verification scripts, 
such as **Playwright** [4]. This ensures the agent confirms its own 
work—simulating user movements and checking UI states—before declaring a task 
complete [4, 15].
8.  **Disable "Fast Mode" to Save Credits:** Turn off "Fast Mode" when using 
high-tier models like GPT 5.5 to prevent burning through your usage quota 2.5 
times faster [16, 17]. Tù Bà Khuỳm recommends this for non-urgent tasks to 
maximize the longevity of your credits [16].
9.  **Reset Context via New Chats:** To effectively clear your context window 
and avoid "autocompaction" issues, start a new chat rather than using a `/clear`
command [18]. Chase AI notes that each new thread acts as a fresh session, which
is essential for managing the 258K token limit in 5.5 Pro [18].
