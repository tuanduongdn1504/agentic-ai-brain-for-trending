## 5. Routine v2 refinement suggestions (minor, not blocking)

### A. Origin-story probe enhancement

**Need:** TrendRadar 52.1K with no documented origin. v2 probe should detect "no origin documented" as signal distinct from "community-viral origin documented".

**Proposed refinement:** Phase 0 origin-story field → values: documented (individual / methodology / community-viral) OR unknown-no-documentation.

### B. Regional-ecosystem classification (v2.1 candidate)

**Need:** TrendRadar revealed CN-ecosystem has parallel infrastructure. v2 doesn't systematically classify regional ecosystem.

**Proposed refinement:** Phase 0 add field "Regional ecosystem" → Western / CN / JP / KR / EU / other / mixed. Would help future Pattern #18 regional tests.

### C. License-choice signaling

**Need:** TrendRadar GPL-3.0 = corpus-first copyleft. Currently license is data point but not structural.

**Proposed refinement:** Phase 0 license-category → permissive (MIT/Apache) vs copyleft (GPL/AGPL). Track as Pattern #29 data source.

### D. External-reference catalog fetch

**Need:** Multiple corpus wikis (spec-kit 30+, agency-agents 144, TrendRadar 11+) reference external docs. v2 "external URL resolution" was noted but not used.

**Proposed refinement:** Phase 2 mandate: if README references catalog/integration guide externally, WebFetch that doc as Phase 2 sub-summary.

### Backlog

4 minor refinements for future v2.1. Not blocking — v2 works well in practice.
