## Definition

`anthropics/skills` is **Anthropic PBC's authoritative reference implementation of the Agent Skills standard** (external standard at agentskills.io), structured as a public repository containing the **SKILL.md spec definition** at `./spec/`, a **template-skill** at `./template/`, and **100+ example skills** organized across **4 categories** (Creative & Design + Development & Technical + Enterprise & Communication + Document Skills), distributed via Claude Code Plugin Marketplace as **2 sub-plugins** (`document-skills` source-available + `example-skills` Apache-2.0), available across **4 channels** (Claude Code Marketplace + Claude.ai auto-availability + Claude API + direct install commands), licensed under **CORPUS-NOVEL mixed-licensing within single repo** (Apache-2.0 for many skills + source-available for docx/pdf/pptx/xlsx), with **top-level THIRD_PARTY_NOTICES.md** artifact, **skills.sh external metadata-tracking service** badge, **Notion partner-skill ecosystem** precedent, and **educational-demonstration disclaimer** Pattern #83 within-pattern strengthening.

## Why this is a single entity

The 100+ example skills + SKILL.md spec + template-skill + 2-plugin sub-marketplace are **deliberately packaged as the foundational-vendor's reference implementation** under one repository with shared spec authority + shared template + mixed-licensing differentiation only between document-skills and example-skills sub-plugins. Treating sub-components as separate entities would lose the **vendor-authoritative reference-implementation identity** which is the load-bearing property.
