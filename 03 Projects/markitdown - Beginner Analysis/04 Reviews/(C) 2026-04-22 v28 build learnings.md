## What needs improvement

### 1. Audit trigger cadence = 1 wiki

**Problem:** v27 audit cleared 1.00:1 → 0.93:1. v28 pushes to 1.07:1. Audit-to-trigger gap = 1 wiki.

**Root cause:** recent wikis produce 3-5 candidates each; audit only retires ~4 per pass. Net positive accumulation.

**Action item for v2.1:**
- **Pre-registration candidate review** — before registering new candidate, verify it's not overlap with existing
- **Stricter novelty threshold** — require 2+ distinguishing features vs existing patterns
- **Parallel promotion review** — each audit should actively promote, not just retire
- **Mini-audit at 0.95:1** — rebalance before 1.00:1 hit

### 2. Pattern #17 variant sub-classification growing complex

**Problem:** Pattern #17 now has 6 variants at N≤2. Sub-classification is becoming N-ary taxonomy within single pattern.

**Action item for v2.1:**
- Consider split: Pattern #17 + Pattern #17a-f as sub-patterns
- Or keep flat: Pattern #17 with growing variant table (current)
- Audit should formalize at v29+

### 3. Corporate-amplified vs organic-viral ambiguity

**Problem:** Pattern #27 Community-Viral Scale Path now spans 6 data points including Microsoft corporate-amplified (114K). But "community-viral" implies organic; Microsoft is corporate-distribution.

**Action item for v2.1:**
- Sub-classify #27: #27a organic-community-viral vs #27b corporate-amplified
- Or tighten #27 to organic-only + register new pattern for corporate-amplified

### 4. Microsoft under-represented in corpus

**Problem:** 2 Microsoft entries (v6 + v28). Actual Microsoft AI ecosystem much larger (AutoGen, Semantic Kernel, Copilot Studio, VS Code AI, Magentic-One, ...).

**Action item for future wikis:** Microsoft AutoGen framework itself is high-priority future candidate (would validate variant 7 AutoGen-extension-ecosystem pattern at N=2).

### 5. Pattern candidate #62 hashtag discovery low-promotion-likelihood
