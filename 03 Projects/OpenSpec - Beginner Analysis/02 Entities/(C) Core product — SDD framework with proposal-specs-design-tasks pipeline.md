## 1. Identity

**Name:** OpenSpec
**Owner:** Fission-AI (org)
**Package:** `@fission-ai/openspec` (npm; v1.3.1 at probe 2026-05-07)
**License:** MIT
**Tagline:** "Spec-driven development (SDD) for AI coding assistants"

## 2. What it does

OpenSpec is a CLI scaffolder + skill/command-file generator that introduces a **specification layer between human-AI dialogue and code generation**. The intent: stop AI coding assistants from "vague prompts and unpredictable results" by formalizing the proposal → specs → design → tasks artifacts before any implementation.

## 3. Pipeline architecture

```
[/opsx:propose <feature>]
       ↓
   change-folder/
   ├── proposal.md     (rationale + scope)
   ├── specs/          (requirements specifications)
   ├── design.md       (technical approach)
   └── tasks.md        (implementation checklist)
       ↓
[/opsx:apply]          (executes tasks; tracks progress)
       ↓
[/opsx:archive]        (moves completed → archive; updates specs canonical)
```

## 4. 11 generated skill commands

Each supported tool receives 11 skill/command files generated by OpenSpec:

`openspec-propose / openspec-explore / openspec-new-change / openspec-continue-change / openspec-apply-change / openspec-ff-change / openspec-sync-specs / openspec-archive-change / openspec-bulk-archive-change / openspec-verify-change / openspec-onboard`

Files deployed as `openspec-*/SKILL.md` (skills installation) OR per-tool command formats (markdown / TOML / prompt format).
