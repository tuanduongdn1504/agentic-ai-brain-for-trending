## PRIMARY — "Personal Daily-Driver Skill Collection (creator-curated, eclectic)" N=1

### The model
khazix-skills is, in the author's own words, *"a few AI skills I actually use every day, open-sourced as-is… running in my own projects long enough to prove it saves time before I bothered open-sourcing it."* The organizing principle is **personal daily use**, not a domain or a function matrix — the 5 skills are deliberately **heterogeneous** (dev-workflow sync / research-report generation / personal writing-voice / AI-news query / disk cleanup), unified only by "the creator personally uses these."

- **≠ T1 Domain-Vertical-Skill-Collection** (CONFIRMED; v64 SEO, v90 academic, v98 cyber, v119 nature, v124 scientific) — those cover *one* domain comprehensively. khazix spans 5 unrelated domains.
- **≠ v121 CodexKit "Multi-Domain (Cross-Functional) Skill Collection"** — CodexKit is a *structured business-ops matrix* (90 skills × 15 work-domains), Codex-native. khazix is a *small personal grab-bag* (5 skills), Claude/agentskills.io-native, with no functional matrix.
- **≠ v81 taste-skill "Multi-Skill Brand Portfolio"** — taste-skill is a *designed product portfolio* (a brand with a sub-variant taxonomy). khazix is *use-then-share*: tools the creator already runs daily, published as-is.

The nameable model: **a creator open-sources the exact eclectic set of skills they personally use every day** (a daily-driver toolkit), distinct from a curated library or an ops matrix.
