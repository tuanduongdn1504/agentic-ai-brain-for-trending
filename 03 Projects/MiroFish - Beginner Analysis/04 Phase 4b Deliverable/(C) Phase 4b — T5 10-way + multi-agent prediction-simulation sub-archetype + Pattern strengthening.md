## Part 8 — i18n architecture: aspirational-vs-actual mismatch

1. **LLM-instruction layer**: 7 languages registered with `llmInstruction` field (zh / en / es / fr / pt / ru / de) — full
2. **UI string layer**: 2 of 7 actually translated (en + zh) — partial 28%
3. **Documentation layer**: 2 of 2 translated (en + zh README) — full at scope

### Comparison to corpus i18n leaders

| Wiki | LLM-steering | UI strings | README | Notes |
|------|--------------|-----------|--------|-------|
| LlamaFactory v22 | n/a | n/a | many | Documentation-heavy localization |
| DeepTutor v38 | inferred | inferred | 9 languages | Strongest README localization at T5 |
| oh-my-claudecode v27 / fish-speech v20 | n/a | n/a | 7 languages | README-only multi-lang |
| awesome-mcp-servers v31 | n/a | n/a | 7 languages | README-only |
| **MiroFish v49** | **7 (full)** | **2 of 7 (partial)** | **2 (full)** | **Only corpus wiki with explicit LLM-steering layer + partial UI translation** |

### Architectural insight

MiroFish has **3 separate i18n decisions** that other corpus wikis don't decompose:
- LLM-instruction layer: cheap to add (one string per language); MiroFish supports 7
- UI string layer: expensive to translate (38 KB en.json equivalent per language); MiroFish supports 2
- Documentation layer: medium-cost; MiroFish supports 2

**Lesson** (not new pattern, observational): aspirational i18n at one layer ≠ implemented i18n at another. **Don't claim "supports 7 languages" without specifying layer.**

## Closing observation

MiroFish v49 is the corpus's first **multi-agent simulation** application — **not** a multi-agent **orchestration** framework like paperclip or rowboat. The distinction matters:

- **Orchestration**: many agents work together to accomplish one task (build a feature, write a report, browse a website)
- **Simulation**: many agents represent a synthetic society used to predict outcomes of real-world inputs

Storm Bear's vault is built on **simulation** of a different sort — Karpathy's LLM Wiki pattern is "simulating what a knowledge worker would do" in code. MiroFish's contribution to the corpus is showing that **agent-based simulation as a product category** is a real and well-funded niche, distinct from agent-based orchestration. This is corpus-broadening, not corpus-deepening.

**The vault's own meta-pattern (LLM Wiki) is now reflected in 3 corpus archetypes**: Karpathy-individual (autoresearch), graphify-Karpathy-inspired-skill (graphify v16), and now agent-society-simulation as a product domain (MiroFish v49). The pattern is robust across product genres.
