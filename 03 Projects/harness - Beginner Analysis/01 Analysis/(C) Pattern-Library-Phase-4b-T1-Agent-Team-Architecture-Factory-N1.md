## PRIMARY — NEW T1 sub-archetype: "Agent-Team-Architecture Factory" (PROVISIONAL N=1, CORPUS-FIRST)

**Definition.** A Claude Code skill/plugin whose **output is a coordinated multi-agent *team***: it synthesises, from a one-sentence domain description, (1) **agent definitions** (`.claude/agents/{name}.md`, with roles + I/O + team-communication protocols), (2) the **skills** those agents use (`.claude/skills/{name}/SKILL.md`), (3) an **orchestrator** that wires data-flow + error-handling + execution-mode, and (4) a **`CLAUDE.md` pointer** — selected via named **team-architecture patterns** (Pipeline / Fan-out-Fan-in / Expert-Pool / Producer-Reviewer / Supervisor / Hierarchical-Delegation) and an execution-mode decision (Agent Teams / Subagents / Hybrid). The author's own framing: an **"L3 Meta-Factory / Team-Architecture Factory"** — the layer that *generates* harnesses rather than being one.

**5-property structural test (all PASS at source):**
1. **Output = a team, not a single artifact** — multiple agents + their skills + orchestrator, not one skill (contrast skill-creator).
2. **Per-domain synthesis** — bespoke team generated from a domain sentence (contrast a fixed shipped team).
3. **Named team-coordination patterns** — 6 patterns + mode decision tree drive the structure.
4. **Self-evolving** — Phase 7 + operate/maintain branch update the generated team over time; change-log discipline in `CLAUDE.md`.
5. **Built on a team primitive** — rides Claude Code's experimental Agent Teams (`TeamCreate`/`SendMessage`/`TaskCreate`).

**Why it is distinct from the nearest corpus subjects (the anti-conflation check):**
| Subject | What it does | Why ≠ harness |
|---|---|---|
| **v95 skill-creator** (anthropics) | generates a *single* skill | harness generates a *team* (many agents + skills + orchestrator) |
| **v126 compound-engineering** | *ships a fixed* 37-skill/51-agent team as its product | harness *generates a bespoke team per domain*; it has no fixed team of its own (its `_workspace/` is just a self-demo) |
| **v94 Understand-Anything** | *runs* a fixed 7-agent pipeline at execution time | harness *authors* the team artifacts; it is a factory, not a runtime |
| **v107 claude-code-harness** (Chachamaru) | an Enforced-Gate *operating loop* | a different shape (a loop, not a team-generator); same word "harness", different author/repo |
