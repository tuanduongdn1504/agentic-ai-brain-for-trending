### Why codymaster chose EN-primary despite VN authorship

Inferred reasons (not confirmed with author):
- **Market reach** — English-speaking developer community >> Vietnamese
- **Documentation discoverability** — GitHub SEO, npm keywords, technical audience default EN
- **Framework name + skill names in EN** — "cm-tdd", "cm-debugging" already EN tokens
- **Contributor onramp** — global contributors expect EN baseline

### Why machine-translate VN when author is VN

- **Time economy** — hand-authoring VN README parallel to EN = 2× writing cost
- **Consistency** — machine translation updates systematically when EN updates
- **Priority decision** — 6 languages (EN/VI/ZH/RU/KO/HI) — only EN gets author effort; all others machine

### What would VN-primary look like (hypothetical)

If codymaster were VN-primary:
- README.md would be in VN
- Skill names would be `cm-` prefixed but potentially VN descriptors
- Examples would use VN product context (Shopee/Tiki/VNG/local fintech)
- Community would include Zalo/Facebook VN group
- Discord would have `#tiếng-việt` channel

→ **None of this exists.** codymaster is cleanly EN-primary with VN afterthought.

## 11. Storm Bear relevance (PRIMARY)

### Why this entity earns its own page

Storm Bear operator = Vietnamese + software dev + Scrum coach + product-adjacent. For 11 of 12 projects analyzed, author is culturally foreign (US solo / US LLC / ByteDance / Microsoft / Karpathy / YC). codymaster is the **first 1-to-1 cultural-demographic match** between Storm Bear's profile and a framework author.

### Concrete Storm Bear benefits

1. **Peer-category intuition transfer** — whatever mental model Tody Le brings to AI-assisted shipping likely overlaps Storm Bear's. Adopting codymaster = borrowing a peer's thinking apparatus.
2. **VN market context sensitivity** (latent) — even without explicit VN examples, the author's background means design decisions may implicitly reflect VN-dev constraints (budget-sensitive infra, mobile-first, fragmented ecosystem)
3. **Community contribution pathway** — Storm Bear could propose VN-specific skills (e.g., `cm-zalo-deploy`, `cm-vnpay-checkout`) with meaningful chance of acceptance from a VN author

5. **"Can't code" affinity** — Storm Bear's Scrum-coach + dev hybrid matches Tody Le's product-lead + ships-via-AI hybrid. Framework designed for that hybrid.

### Concrete Storm Bear risks

1. **Single-author bus factor** — if Tody Le stops maintaining, no LLC/corporate fallback
2. **38 stars** — tiny community, limited empirical peer data
3. **VN quality still machine-level** — peer author doesn't solve translation quality
4. **No VN community signal** — may need Storm Bear to create, not just join
5. **Breaking changes** — solo author can refactor freely without community backlash
