## The 5-mechanic methodology

### Mechanic 1 — Core Asset Protocol (mandatory for brand-specific work)

5-step process:
1. **Ask** — clarify brand assets needed for the design task
2. **Search** — find existing brand assets (logo, colors, typography, voice)
3. **Download** — pull verified assets from sources
4. **Verify** — confirm asset authenticity (color exactness, logo version, font licensing)
5. **Freeze to spec** — lock assets into design spec for downstream consistency

**Design philosophy** (verbatim): *"great hi-fi design doesn't start from a blank page, it grows from existing design context."*

### Mechanic 2 — Design Direction Advisor

Recommends **3 differentiated directions** drawn from a **5 schools × 20 philosophies = 100-philosophy** taxonomy. Activated when briefs are vague (low-signal user request).

### Mechanic 3 — Junior Designer Workflow

Show-early-iteration discipline:
- Show work early with assumptions + placeholders
- Iterate on content
- Iterate on variations
- Iterate on tweaks

= Iterative-display discipline preventing single-pass output.

### Mechanic 4 — Fact Verification First

WebSearch-grounded confirmation of products/technologies mentioned in design tasks, rather than relying on LLM training-data alone. Prevents hallucinated brand/product details in design output.

### Mechanic 5 — Anti AI-slop Rules

**CSS-technique enumeration** (verbatim):
- `text-wrap: pretty`
- CSS Grid (vs flexbox-for-layouts overuse)
- "carefully chosen serif display faces" (vs default sans-serif AI defaults)
- `oklch` colors (vs RGB/HSL AI defaults)

**Avoided visual clichés** (verbatim cliché enumeration):
- Purple gradients
- Emoji icons
- Rounded corners with left borders
- SVG silhouettes
